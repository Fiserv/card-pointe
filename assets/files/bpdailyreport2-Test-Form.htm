Step 1 - Fill In Data
SECRET_KEY: TPS1
ACCOUNT_ID: TPS2
12-digit Bluepay 2.0 Gateway Account ID
REPORT_START_DATE: YYYY-MM-DD HH:MM:SSTPS3
REPORT_END_DATE: YYYY-MM-DD HH:MM:SSTPS4
RESPONSEVERSION:
Optional. 1 or 2 (Defaults to 1)
1 or not set = Version 1 result fields retruned.
2 = All version 1 result fields returned plus addtional version 2 result fields.
3 = All version 2 result fields returned plus addtional version 3 result fields.
4 = All version 3 result fields returned plus addtional version 4 result fields.
DO_NOT_ESCAPE:
Optional. 0 or 1 (Defaults to 0)
0 or not set = values are quoted.
1 = The output will have all commas and quotes removed from the data and to be simply comma-separated.
QUERY_BY_SETTLEMENT:
Optional. 0 or 1 (Defaults to 0)
0 or not set = Query by transaction date
1 = Query by settlement date
EXCLUDE_ERRORS:
Optional. 0 or 1 (Defaults to 0)
0 or not set = Include errors
1 = Exclude errors
QUERY_BY_HIERARCHY:
Optional. 0 or 1 (Defaults to 0)
0 or not set = Do not include transaction data from child accounts
1 = Include transaction data from child accounts
MODE:
Optional. TEST or LIVE (Defaults to LIVE)
PAYMENT_TYPE:
Optional. Valid values CREDIT, ACH or SEPA
ORIGIN:
Optional.
TPS_HASH_TYPE:
The algorithm used to compute the TAMPER_PROOF_SEAL. Accepted values are 'MD5', 'SHA256', 'HMAC_SHA256', 'SHA512' or 'HMAC_SHA512'(preferred).
TAMPER_PROOF_SEAL:

MD5 hash of Secret Key and transaction parameters
